ภาพนี้เป็นหน้าจอในส่วนของ ผู้ดูแลระบบ (Admin) เมื่อผู้ดูแลระบบเข้าสู่ระบบเรียบร้อยแล้ว ให้คลิกที่ไอคอนโปรไฟล์ (Profile) ที่มุมขวาบนของแถบเมนู ระบบจะแสดงเมนูแบบเลื่อนลง (Dropdown Menu) ซึ่งประกอบด้วย:
•	บัญชีของฉัน
•	Dashboard
•	Logout
โดยเมนู Dashboard เป็นเมนูที่เพิ่มเข้ามาเฉพาะสำหรับผู้ดูแลระบบ
![](../../image2/Blacklist(1).pdf)
ภาพนี้แสดงหน้าแดชบอร์ดของผู้ดูแลระบบ ซึ่งเป็นหน้าศูนย์กลางสำหรับบริหารจัดการข้อมูลภายในระบบ โดยผู้ดูแลสามารถเข้าถึงเมนูจัดการต่าง ๆ ได้จากหน้านี้
หน้าแดชบอร์ดแบ่งออกเป็น 3 ส่วนหลัก ได้แก่
1.	แถบเมนูด้านบน (Top Navigation Bar)
2.	เมนูจัดการด้านซ้าย (Sidebar Menu)
3.	พื้นที่แสดงผลข้อมูลหลัก (Main Content Area)
1.บริเวณด้านบนของหน้าจอประกอบด้วย:
•	ชื่อระบบ (เช่น PNNAdmin)
•	ไอคอนแจ้งเตือน (Notification)
•	ชื่อบัญชีผู้ดูแลระบบ (เช่น System)
•	ไอคอนโปรไฟล์ผู้ใช้งาน
จุดประสงค์ของส่วนนี้คือแสดงสถานะการเข้าสู่ระบบ และให้ผู้ดูแลสามารถเข้าถึงเมนูบัญชีหรือออกจากระบบได้อย่างรวดเร็ว
2.แถบเมนูด้านซ้ายเป็นส่วนสำคัญสำหรับการบริหารจัดการระบบ โดยประกอบด้วยเมนูต่าง ๆ ดังนี้
•	User Management (จัดการผู้ใช้งาน)
•	Vehicle Management (จัดการยานพาหนะ)
•	Route Management (จัดการเส้นทาง)
•	Booking Management (จัดการการจอง)
•	Driver Verification Management (จัดการการยืนยันตัวตนผู้ขับ)
•	User Blacklist Management (จัดการบัญชีที่ถูกระงับ)
•	Home (กลับสู่หน้าหลักเว็บไซต์)
•	Setting (ตั้งค่าระบบ)
ผู้ดูแลสามารถคลิกแต่ละเมนูเพื่อเข้าสู่หน้าจัดการข้อมูลตามประเภทนั้น ๆ
3.จากภาพพื้นที่แสดงผลข้อมูลหลัก ตัวอย่างที่แสดงอยู่คือหน้า “User Management” ซึ่งมีองค์ประกอบดังนี้
3.1 ปุ่มสร้างผู้ใช้ใหม่
มีปุ่ม “สร้างผู้ใช้ใหม่” สำหรับเพิ่มบัญชีผู้ใช้งานเข้าสู่ระบบ
3.2 ระบบค้นหาและกรองข้อมูล
ผู้ดูแลสามารถค้นหาผู้ใช้โดย:
•	Email
•	Username
•	ชื่อ–นามสกุล
และสามารถกรองข้อมูลตาม:
•	วันที่เริ่มต้น – วันที่สิ้นสุด
•	สถานะผู้ใช้งาน
•	การเรียงลำดับข้อมูล
3.3 ตารางแสดงข้อมูลผู้ใช้งาน
ตารางแสดงข้อมูลสำคัญของผู้ใช้แต่ละราย เช่น
•	ชื่อผู้ใช้
•	อีเมล
•	บทบาท (Passenger / Driver)
•	สถานะการยืนยันตัวตน
•	สถานะการใช้งาน (Active / Inactive)
•	วันที่สร้างบัญชี
•	ปุ่มการกระทำ (ดูรายละเอียด / แก้ไข / ลบ)
ผู้ดูแลสามารถเปิด–ปิดสถานะผู้ใช้งานได้จากปุ่มสวิตช์ และดำเนินการแก้ไขหรือลบบัญชีได้ทันทีจากตาราง
![](../../image2/Blacklist(2).pdf)
ในหน้า User Management ผู้ดูแลระบบสามารถดำเนินการ Blacklist ผู้ใช้งานได้จากเมนูการกระทำ (Action) ภายในตารางรายชื่อผู้ใช้
ภายในตารางรายชื่อผู้ใช้งาน จะมีคอลัมน์ “การกระทำ” ซึ่งแสดงไอคอนต่าง ๆ เช่น
•	ไอคอนรูปตา (ดูรายละเอียด)
•	ไอคอนแก้ไข
•	ไอคอนลบ
เมื่อเข้าสู่หน้ารายละเอียดผู้ใช้ ระบบจะแสดงข้อมูลดังนี้:
1. ข้อมูลส่วนบุคคล
•	อีเมล
•	ชื่อผู้ใช้
•	เบอร์โทรศัพท์
•	บทบาทในระบบ (Passenger / Driver)
2. หลักฐานการยืนยันตัวตน
•	เลขบัตรประชาชน
•	ชื่อ–นามสกุลจริง
ข้อมูลส่วนนี้มีไว้เพื่อให้ผู้ดูแลตรวจสอบความถูกต้องก่อนดำเนินการใด ๆ กับบัญชีผู้ใช้
![](../../image2/Blacklist(3).pdf)
บริเวณด้านขวาของหน้าจอจะเป็นส่วน “การจัดการสิทธิ์การใช้งาน”
มีข้อความแจ้งเตือนว่า:
การ Blacklist จะตัดสิทธิ์การเข้าถึงทันที และบล็อกข้อมูลสำคัญของผู้ใช้ไม่ให้สมัครใหม่
หมายความว่าเมื่อดำเนินการ Blacklist แล้ว ผู้ใช้งานจะไม่สามารถเข้าสู่ระบบได้อีก และข้อมูลสำคัญจะถูกใช้ป้องกันการสมัครบัญชีใหม่
ก่อนกดปุ่ม “ลงโทษ BLACKLIST ถาวร” ผู้ดูแลต้องกรอกข้อมูลให้ครบถ้วนก่อน ดังนี้
![](../../image2/Blacklist(4).pdf)
 เลือกประเภทความผิด (บังคับกรอก)
ในหัวข้อ “ประเภทความผิด (ตาม ROLE: PASSENGER)”
ผู้ดูแลต้องเลือกประเภทความผิดจากรายการที่ระบบกำหนด เช่น
•	ก่อกวน / ทำลายทรัพย์สินในรถ
•	ค้างชำระค่าบริการ / เรียกแล้วไม่มา
•	ใช้คำพูดไม่สุภาพ / ดูหมิ่นคนขับ
•	พยายามใช้ไอดีคนอื่นเดินทาง
•	แจ้งความเท็จ / ก่อกวนระบบ
•	อื่น ๆ (ระบุเพิ่มเติมด้านล่าง)
ระบบจะไม่อนุญาตให้ดำเนินการ Blacklist หากยังไม่ได้เลือกประเภทความผิด
ผู้ดูแลต้องระบุรายละเอียดพฤติกรรมหรือหลักฐานเพิ่มเติมในช่องข้อความ เช่น
•	รายละเอียดเหตุการณ์
•	วันที่เกิดเหตุ
•	หลักฐานอ้างอิง
ข้อมูลนี้ใช้เป็นบันทึกประกอบการตัดสินใจ และสามารถใช้ตรวจสอบย้อนหลังได้
เมื่อกรอกข้อมูลครบถ้วนแล้ว:
1.	ปุ่ม “ลงโทษ BLACKLIST ถาวร” จะสามารถกดได้
2.	เมื่อกด ระบบจะดำเนินการ:
o	ระบบจะแจ้งเตือนการยืนยัน เพื่อให้ผู้ดูแลยืนยันการ blacklist อีกครั้ง
o	เปลี่ยนสถานะผู้ใช้เป็น Blacklist
o	ปิดการใช้งานบัญชีทันที
o	บันทึกเหตุผลและรายละเอียดลงในระบบ
หลังจากดำเนินการเสร็จ ผู้ใช้จะไม่สามารถเข้าสู่ระบบได้อีก
![](../../image2/Blacklist(5).pdf)
ตารางแสดงรายชื่อผู้ใช้ที่ถูกระงับ
ด้านล่างเป็นตารางแสดงรายละเอียดผู้ใช้งานที่ถูก Blacklist โดยประกอบด้วยข้อมูลสำคัญดังนี้:
1. User Profile
แสดงข้อมูลพื้นฐานของผู้ใช้ ได้แก่:
•	รูปโปรไฟล์ (หรืออักษรย่อ)
•	ชื่อผู้ใช้
•	Username (@example)
•	บทบาทในระบบ (Passenger / Driver)
2. Reason (เหตุผลที่ถูก Blacklist)
แสดงประเภทความผิดและรายละเอียดที่ผู้ดูแลกรอกไว้ในขั้นตอนการ Blacklist เช่น:
•	ค้างชำระค่าบริการ / เรียกแล้วไม่มา
•	ก่อกวน / ทำลายทรัพย์สิน
•	ใช้คำพูดไม่สุภาพ
•	รายละเอียดเพิ่มเติมที่ระบุในระบบ
ข้อมูลส่วนนี้ช่วยให้สามารถตรวจสอบสาเหตุของการลงโทษย้อนหลังได้

เอกสารคู่มือการใช้งานระบบ (User Manual) ฉบับนี้ ได้มีการใช้เครื่องมือปัญญาประดิษฐ์ (Artificial Intelligence: AI) เพื่อช่วยสนับสนุนในการเรียบเรียงเนื้อหาและปรับปรุงรูปแบบภาษาให้มีความชัดเจน เป็นทางการ และเหมาะสมกับรูปแบบเอกสารทางวิชาการ

•	จัดรูปแบบเนื้อหาให้อยู่ในลักษณะของคู่มือการใช้งาน (User Manual)
